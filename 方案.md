# Novel Voice Cast — 开发方案

## 背景

我喜欢看轻小说，但轻小说的对话通常不标注说话人，只用「」包裹。市面上的 TTS 工具要么只有单一朗读者，要么配置复杂、需要自己折腾音色克隆。

这个项目的目标很简单：**上传一本已标注好说话人的小说 → 给每个角色分配音频（用户上传 or 预设音色）→ 一键合成带角色演绎的有声书**。

整个小说转语音大项目分为三个阶段：
1. **novel_correct** — OCR 纠错，统一「」符号
2. **opencode-novel-loop** — 利用 AI 自动标注每句对话的说话人
3. **Novel Voice Cast（本项目）** — 最终阶段：Web UI + 音色克隆/TTS，输出有声书

本方案覆盖的是**第三阶段**，也是连接用户与有声书的最后一公里。

> **核心原则：** 不重复造轮子。opencode-novel-loop 已经实现了成熟的 LLM Agent 架构，我们直接复用其核心组件。

## 一句话流程

```
labeled_novel.txt ──→ 解析提取对话+角色 ──→ Web UI 分配音频 ──→ 音色克隆+TTS ──→ 输出完整有声书
```

解析是一瞬间的事，核心全部在 UI + 合成调度上。

## 技术选型

### 后端

| 模块 | 选型 | 理由 |
|------|------|------|
| Web 框架 | **FastAPI** | 异步友好，自带 OpenAPI 文档 |
| 数据库 | **SQLite + SQLAlchemy** | 零配置，ORM 顺手 |
| 音频处理 | **pydub** + **soundfile** | 格式转换、拼接 |
| 音色克隆 | **你先找，我留好适配接口** | 专为一对一对接设计 |
| 预设 TTS | **edge-tts** | 免费，中文自然，一行代码出音频 |
| 异步任务 | **ARQ** (Redis) | 轻量异步任务队列 |
| 文件存储 | **本地文件系统** | 简单直接 |

### 前端

| 选型 | 理由 |
|------|------|
| **Vue 3 + Vite** | 开发体验好，热更新快 |
| **Naive UI** | 组件丰富，中文友好 |
| **Pinia** | 状态管理 |
| **Vue Router** | 路由 |
| **wavesurfer.js** | 音频波形 + 播放控制 |

> | 用克隆音色合成对白 | 调用 `model.generate(text, reference_wav_path)` |
> | 无参考音频的角色 | 走原来的 edge-tts 预设，两者并存 |
> | 语气/情绪控制 | 在文本前加 `(欢快活泼)` 等描述即可 |

## 数据格式说明

### 输入 labeled_novel.txt 支持的格式

```
「你终于来了。」——绫小路清隆
绫小路清隆：「嗯，路上有点事。」
「那就好。」——绫小路清隆

旁白：教室里一片寂静。
```

### 角色音频映射

```
角色 → { source: "upload", audio_id: "uuid" }
角色 → { source: "preset_male" }
角色 → { source: "preset_female" }
```

未分配的角色：根据性别自动分配预设音色。
